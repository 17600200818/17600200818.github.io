api的restful设计原则

crud和get post put delete对应关系

c =》 get 查
r =》 post 增
u =》 put 更新
d =》 delete 删除

例子

创建用户

/user method post
201 创建成功
400 请求错误
500 服务器内部错误

用户登陆

/user/:username post
200 ok
400 请求错误
500 服务器内部错误

获取用户信息

/user/:username get
200 ok
400 请求错误
401 用户认证失败
403 用户认证成功但是没有权限
500 服务器内部错误

注销用户

/user/:username delete
204 删除成功
400 请求错误
401 用户认证失败
403 用户认证成功但是没有权限
500 服务器内部错误

爬虫小记-爬取sql查询平台sql执行结果

背景

因为外部门有一个sql查询平台，需要做一个工具直接运行sql后取得csv数据，因为不能在外部门平台上开发，即爬取外部门网站运行sql后的结果自己做处理

整个过程遇到的问题

1.登陆cookie问题

网站登陆一次后一直使用一个cookie，所以在请求头中加入cookie即可

2.csrf跨站请求限制

因为网站采用了csrf验证限制，所以在请求网站的时候网站源码中会存在csrf的值，在抓取之前先爬取一下网页的内容，然后根据正则来筛选出来csrf_token在下次请求访问接口的时候带上即可

总结

编写爬取代码之前先用抓包工具或者postman来访问，预估问题解决后再开动
抓取网页前需要认真分析需要的接口参数，没有线索可以去js文件或者网页源代码中找，多尝试用工具调试api，网页爬虫一般都比较容易

javascript的Promise对象使用

Promise

Promise 对象用于表示一个异步操作的最终状态（完成或失败），以及其返回的值。
对象由关键字 new 及其构造函数来创建。构造函数会把一个处理器函数作为参数。这个处理器函数接受两个函数resolve 和 reject。当异步任务顺利完成且返回结果值时，会调用 resolve，而当异步任务失败且返回失败原因（通常是一个错误对象）时，会调用reject 函数。
